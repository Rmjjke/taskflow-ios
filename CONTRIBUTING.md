# Contributing to TaskFlow iOS

## Development Workflow

### 1. Branch

Always cut feature branches from `develop`, never from `main`.

```bash
git checkout develop
git pull origin develop
git checkout -b feature/US-01-quick-capture
```

Branch naming:
- Features: `feature/US-XX-short-description`
- Bugs: `fix/short-description`
- Chores: `chore/short-description`

---

### 2. Build Setup

```bash
# Install tooling (once)
brew install xcodegen swiftlint xcbeautify

# Generate project and open
xcodegen generate
open TaskFlow.xcodeproj
```

> **Important:** Never commit `TaskFlow.xcodeproj/`. It is generated by XcodeGen and lives in `.gitignore`. All project structure changes go into `project.yml`.

---

### 3. Code Standards

#### Architecture rules
- ViewModels must be `@Observable` and `@MainActor`.
- Views must never import `SwiftData` directly — all data access goes through the `TaskRepositoryProtocol`.
- New screens follow the `XxxView` / `XxxViewModel` naming pair.

#### Swift style
- Swift 6, strict concurrency — no `@unchecked Sendable`, no `nonisolated` workarounds without justification.
- `SwiftLint` must pass with zero errors before opening a PR.
- No force-unwraps (`!`) in production code — use `guard let` or `if let`.
- `updatedAt` must be set on every `TaskItem` mutation (CloudKit sync requirement for v1.0).

---

### 4. Tests

Every story must include:

| Type | Coverage |
|---|---|
| Unit test | All `TaskRepository` mutations |
| Unit test | ViewModel state transitions |
| UI test | Happy path for the core user interaction |

Run tests locally before pushing:

```bash
xcodebuild test \
  -scheme TaskFlow \
  -destination "platform=iOS Simulator,name=iPhone 15 Pro,OS=17.5" \
  -only-testing:TaskFlowTests \
  | xcbeautify
```

---

### 5. Pull Request Checklist

Before requesting review, confirm all of the following:

- [ ] Branch is up to date with `develop`
- [ ] `xcodegen generate` succeeds cleanly
- [ ] Build succeeds with zero warnings (Release config)
- [ ] All unit tests pass
- [ ] SwiftLint passes with zero errors
- [ ] New code has corresponding unit tests
- [ ] VoiceOver accessibility tested for any new UI
- [ ] Dark mode verified
- [ ] Tested on at least: iPhone SE (small) and iPhone 15 Pro Max (large)
- [ ] `CHANGELOG.md` updated (if applicable)
- [ ] PR title follows format: `[US-XX] Short description`
- [ ] PR links to the GitHub Issue

---

### 6. Commit Message Format

```
[US-XX] Short imperative description (≤72 chars)

Optional longer body explaining the why, not the what.
Wrap at 72 characters.

Refs #<issue-number>
```

Examples:
```
[US-01] Add QuickAddView with auto-focused title field

[US-03] Animate checkbox completion with spring effect

Refs #12
```

---

### 7. Spec-First Development

> **Never write code without reading the spec first.**

1. Read the relevant story in `docs/epic-task-management.md`
2. Check the PRD section in `docs/prd-taskflow.md`
3. Implement against the Acceptance Criteria
4. Write tests that map to the Definition of Done checkboxes

---

## Release Process

1. Cut a `release/vX.X` branch from `develop`
2. Bump `MARKETING_VERSION` in `project.yml`
3. Run full regression suite on physical devices
4. Merge `release/vX.X` → `main` + tag `vX.X.0`
5. Merge `main` back → `develop`
